분석 전 가정

처음 제출한 기획안은 명함·청첩장 인쇄 주문 전 오타·문구 검수에 초점을 두었습니다.

1
기존 안인쇄 주문 직전 텍스트를 추출하고 사용자가 다시 확인하는 흐름
2
전제명함·청첩장처럼 오탈자 손실이 큰 인쇄물에서 신뢰를 보완
3
한계하지만 툴디의 주 상품은 인쇄물 자체보다 템플릿 경험에 가깝습니다.

VOC로 다시 본 방향

반복 VOC는 크리에이터가 템플릿과 요소를 등록하고 심사받고 노출되는 과정의 불확실성에 더 많이 쏠려 있었습니다.

따라서 구조 기반 필터는 사용자 탐색 기능이자, 크리에이터 업로드·분류·노출 기준을 정리하는 기능입니다.
이 관점에서 다음으로 확인할 것은 현재 화면이 템플릿을 어떤 기준으로 찾게 하고, 어떤 기준으로 분류하게 만드는지입니다.

미리디: 감성 기준 필터가 전면에 보인다

귀여운, 감성적인, 전문적인 같은 감성어로 템플릿을 좁히게 되어 있습니다.

Opportunity 검색 진입은 강하지만, 템플릿을 어떤 기준으로 좁힐지 한 번에 보이는 장치가 약합니다.
Proposal Point 툴디는 감성어보다 템플릿의 실제 구성 방식으로 필터를 제안합니다.

툴디: 검색창은 있지만 선택 기준은 덜 드러난다

템플릿이 핵심 상품으로 보이지만, 결과를 어떤 기준으로 좁힐지 화면에서 즉시 보이지 않습니다.

따라서 툴디는 경쟁사처럼 감성어를 따라가기보다, 템플릿의 실제 구성 방식으로 선택 기준을 만들 수 있습니다.

MAIN IDEA

개선안 1. 구조 기반 스타일 필터

Core Proposal

왜 이 기능인가?

VOC에서 드러난 태그·카테고리·노출 기준의 혼란을 사용자 화면과 크리에이터 업로드 기준으로 함께 풀어냅니다.

1
사용자 관점원하는 구성의 템플릿을 더 빠르게 찾습니다.
2
크리에이터 관점업로드 시 분류 기준을 이해하고 노출을 예측합니다.
3
운영 관점반려·태그·카테고리 문의를 줄일 수 있는 기준이 됩니다.

필터 옵션은 4개만 둔다

사진 중심인물·제품·웨딩 사진이 주요 구성 요소
일러스트 중심그림·아이콘·장식 요소가 주요 구성 요소
도형·그래픽 중심면·선·도형이 주요 시각 언어
글자 중심문구·제목·정보 전달이 핵심
4개 테마는 템플릿의 첫인상을 결정하는 주요 구성 요소를 기준으로 나눴습니다. 사진, 일러스트, 도형·그래픽, 글자 중 무엇이 중심인지 먼저 구분하면 사용자는 결과를 더 빠르게 좁힐 수 있습니다.
